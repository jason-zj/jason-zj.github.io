Hexo Github搭建博客心得

发表于 2016-11-10   |   分类于 随笔   |  

起因

一直在看别人博客的心得与技术分享，感觉很不错，既跟别人分享了自己的成果，也巩固了自己的技术，同时也提高了自己的写作水平！（滑稽脸）

所以萌生了自己也搭建一个博客的想法，不过咱是技术小白，通过百度、谷歌来学学如何搭建博客吧！

误入歧途

可能用这个标题不太准确，至少不太精确。首先搜到的是

使用Github和Jekyll搭建博客

  • 首先，Window 系统下，使用 RailsInstaller 来安装 Ruby 环境。
  • $ ruby -v 查看是否安装成功。
  • $ gem install jekyll 安装jekyll，竟然失败了！

于是我们的百度之旅又开始了，为毛会失败？百度了一番之后得到结果。
默认源被墙了，换个源就好 。gem source -a http://rubygems.org/换个源，再安装，果然成功了!不过这次百度之后，我看到了很多对jekyll的吐槽！

对windows支持不好。以及生成文件慢。并且大家推荐了Hexo。

Hexo 之旅开始！

我是一个听人劝的人。既然大家力荐Hexo，我还何苦去花费时间去碰jekyll的壁。开始研究（百度）Hexo的搭建方法。

环境：Git、NodeJs、Github

安装好NodeJs后，找一个地方建一个blog的文件夹。如：D:\hexo ,打开git bash 。

安装hexo

$ npm install hexo-cli -g
初始化hexo

$ hexo init
$ npm install
这时候你的hexo文件夹里面已经有很多文件了。

$ hexo g
$ hexo s
当你看到这条信息的时候，浏览器打开http://localhost:4000/，恭喜你，博客搭建完成。
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

更改主题及优化

现在我们的本地博客已经搭建好了，可是不满足现状的我们，总是想追求完美，我选用的是简单大气的NexT主题。

安装NexT主题

$ git clone https://github.com/iissnan/hexo-theme-next themes/next
启用NexT主题。打开站点配置文件。如 d:/hexo/_config.yml，找到 theme 字段。修改为Next.

theme: next

特别注意：冒号与next中间要加一个空格。所有配置文件都要这么做，否则会出错！(我才不会说，我就吃过这个亏！)

其他优化选择你所需要的进行修改。打开NexT 使用文档对照相应配置进行更改。

我个人加了多说、背景js以及一些常规配置。

总结

有些事情想起来难，做起来还是挺简单的。所以想到了就去做，Just Do It ！
